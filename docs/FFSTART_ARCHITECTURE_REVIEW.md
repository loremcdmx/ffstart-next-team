# Ревью архитектуры FF Start

Внутренняя страница `/ffstart-review` собирает в одном месте всю текущую программу: 11 модулей, 36 уроков и 5 игровых пауз. Для каждого пункта она показывает краткий смысл, видео, учебные материалы, формат тренажёра и отдельное решение `Оставить / Под вопросом / Убрать` с комментарием.

Страница не входит в маршрут ученика, закрыта от индексации и не пишет ни прогресс, ни telemetry. Содержание для неё генерируется из актуальных источников курса в `course/ffstart-review-data.json`:

```bash
npm run build:ffstart:review
node scripts/test-ffstart-review-contract.mjs
```

Контракт заново собирает JSON во временный файл и сравнивает его с опубликованным `course/ffstart-review-data.json`. Поэтому изменение манифеста, видео, общего банка или legacy-тренажёра не может незаметно оставить ревью на старом снимке.

Практика считается раздельно: `generatedSpots` — ситуации из общего сгенерированного банка, `legacySpots` — реальные ситуации отдельных RFI/BB-тренажёров, `fullSimulatorLessons` — legacy-уроки со свободной игрой в полном симуляторе. Эти числа не складываются в один двусмысленный показатель.

## Как сохраняются решения

Каждое изменение записывается сразу, без ожидания и без отдельной кнопки сохранения. В браузере поддерживаются три записи в двух механизмах хранения:

1. основная запись в `localStorage`;
2. резервная запись в отдельном ключе `localStorage`;
3. резервная запись в IndexedDB.

У каждой записи есть контрольная сумма. При открытии страницы корректные копии объединяются по времени изменения каждого пункта; повреждённая копия не подменяет рабочую. Изменения из разных вкладок также объединяются по пунктам, поэтому два параллельных решения не заменяют друг друга целым снимком. Очистка хранится как датированная операция и не «воскресает» после импорта старого файла. Исчезнувшие из новой версии курса пункты сохраняются в экспорте как `orphaned`.

Каждая localStorage-копия записывается независимо. Даже если обе localStorage-записи запрещены или переполнены, страница всё равно ставит запись в очередь IndexedDB и показывает фактический результат: три копии, частичное сохранение, только IndexedDB либо полный отказ браузерного хранилища.

Все три браузерные записи принадлежат одному origin и могут быть удалены вместе при ручной очистке данных сайта; запрос persistent storage тоже может быть отклонён браузером. Поэтому перед передачей ревью нужно нажать **«Зафиксировать итог»** и сохранить JSON-файл. Это единственная переносимая контрольная копия, которую можно загрузить кнопкой **«Загрузить решения»** на другом компьютере или в другом браузере. Кнопка **«Скопировать сводку»** готовит человекочитаемый Markdown, но не заменяет JSON для восстановления.

## Проверка перед передачей

```bash
npm run check:ffstart
node scripts/check-static.mjs
node scripts/check-local-routes.mjs
git diff --check
```

В браузере дополнительно проверяются: сохранение решения и комментария после reload, синхронизация двух вкладок, фильтры, экспорт и импорт JSON, отсутствие ошибок в консоли, горизонтального overflow и поломок на desktop и mobile.
